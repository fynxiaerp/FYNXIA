## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server Action → ICP private key | .pfx + decrypted password live only in server RAM during signPdfBuffer |
| DB → tenant rows | RLS isolates document_* tables by clinic_id |
| client ← document_versions | storage_path + cert_pem are column-REVOKEd from authenticated/anon |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03 | Information Disclosure | sign-document.ts (private key) | mitigate | `import 'server-only'`; key loaded in-memory via createAdminClient; never serialized, returned, or logged |
| T-08-04 | Tampering | signed PDF bytes vs stored hash | mitigate | sign the FINAL renderToBuffer bytes only; sha256Hex stored; verifyPdfSignature recomputes on a fresh md |
| T-08-05 | Tampering | document_versions immutability | mitigate | RLS: INSERT-only (WITH CHECK), no UPDATE/DELETE policy → signed versions are forever immutable |
| T-08-06 | Information Disclosure | storage_path / cert_pem columns | mitigate | REVOKE SELECT (storage_path, cert_pem) on authenticated, anon (Pitfall 8) |
| T-08-07 | Information Disclosure | cross-tenant document read | mitigate | RLS clinic_id = get_my_tenant_id() on all three tables; index every clinic_id |
| T-08-08 | Spoofing | wrong cert OID / forge API drift | accept | OID fallbacks + certificateToAsn1 pinned from Phase 7; covered by sign-document.test.ts round-trip |
</threat_model>
