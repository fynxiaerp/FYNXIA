<objective>
Deliver DOC-02 (ICP signing) and DOC-03 (immutable versioning) end-to-end: the
generate→sign→verify Server Actions, the immutable version history, the nodejs PDF download
route, the generation UI, and the new `documentos` module that lets clinical roles (dentist/admin)
reach document generation while keeping read-only roles out of mutations.

The signing flow is ATOMIC (Pitfall 1): fill variables → renderToBuffer → SHA-256 → RSA sign
the FINAL bytes → upload that exact buffer to the private documents-pdf bucket → insert the
document_versions row. No re-render after signing. Filled content is AES-encrypted at rest
(is_content_encrypted=true). Verification is on-demand only (separate action), not on every download.

Purpose: Complete the document engine — the cryptographic + immutability core reused by
Receituário (Phase 12), NFS-e (Phase 15), Teleodonto (Phase 20).
Output: documentos module gating + actions + download route + generator/versions UI.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → generate/sign action | template/patient selection crosses into the action |
| Server Action → ICP private key | .pfx + password in server RAM only during signPdfBuffer |
| client ← signed PDF | download via short-TTL signed URL; raw storage_path never sent |
| DB ← filled content | patient PII in filled content encrypted at rest (AES) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-15 | Elevation of Privilege | read-only role signing/generating | mitigate | assertNotReadOnly() first in generateDocument/signDocument/revise; UI buttons disabled for read-only |
| T-08-16 | Tampering | signed bytes ≠ stored bytes | mitigate | atomic render→hash→sign→upload the SAME buffer; verifyDocumentSignature recomputes sha256 + RSA verify |
| T-08-17 | Information Disclosure | ICP private key leak | mitigate | signing in src/lib/icp/sign-document.ts (server-only); key never returned/logged; admin client server-side |
| T-08-18 | Information Disclosure | filled content PII at rest | mitigate | encrypt(filledContent) via crypto.ts; is_content_encrypted=true |
| T-08-19 | Information Disclosure | storage_path / signed PDF exposure | mitigate | createSignedUrl short TTL; raw path never sent; Cache-Control: no-store on download route |
| T-08-20 | Tampering | editing a signed version | mitigate | RLS no UPDATE/DELETE on document_versions; edit creates a NEW draft version (supersedes_id) |
| T-08-21 | Information Disclosure | cross-tenant document access | mitigate | tenant reads via createClient() RLS; admin client used only for keystore + storage + signed-version write |
</threat_model>
