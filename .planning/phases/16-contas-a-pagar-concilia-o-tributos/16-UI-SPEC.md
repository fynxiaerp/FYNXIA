## Spacing Scale

All spacing is Tailwind utility classes mapping to the existing 8-point scale. No new tokens are introduced.

| Token | Tailwind | px | Usage |
|-------|----------|----|-------|
| xs | `gap-1`, `p-1` | 4px | Icon gaps, badge inner padding |
| sm | `gap-2`, `p-2` | 8px | Inline element spacing, compact filter row gaps |
| md | `gap-4`, `p-4` | 16px | Default card/form field spacing |
| lg | `gap-6`, `p-6` | 24px | Section padding (all `<main>` blocks use `p-6`) |
| xl | `gap-8`, `p-8` | 32px | Layout-level gaps between major sections |
| 2xl | `gap-12` | 48px | Major section breaks (rare; empty-state vertical padding uses `py-16`) |
| 3xl | `gap-16` | 64px | Page-level empty-state top/bottom padding |

Exceptions:
- Touch targets for icon-only action buttons: minimum `size-8` (32px) per Tailwind convention used in existing tables (e.g. `MoreHorizontal` triggers in `OsTable.tsx`).
- Table rows: no explicit height; row padding is `px-4 py-3` (cells), matching existing `TransactionList` and `ReceivablesTable`.
- `PageHeader` is fixed at `h-16` (64px) вЂ” do not alter.

**Semantic status colors (NOT accent вЂ” applied directly as Tailwind utilities):**

| Status | Badge classes | Context |
|--------|--------------|---------|
| `pendente` | `bg-amber-100 text-amber-800 border-amber-200` | Payable not yet due, statement line not reconciled |
| `parcial` | `bg-amber-100 text-amber-800 border-amber-200` | Partial payment received |
| `pago` / `conciliado` | `bg-green-100 text-green-800 border-green-200` | Paid / reconciled |
| `vencido` | `bg-red-100 text-red-800 border-red-200` | Overdue (derived at read-time, same as receivables D-04) |

| `rascunho` | `variant="outline"` (shadcn Badge) | RPA or payout draft state |
| `emitido` / `transmitido` | `variant="default"` | RPA emitted / Reinf transmitted |
| Exact auto-match (conciliaГ§ГЈo) | `bg-green-50 border-green-200` row highlight | Stage 1 exact match in reconciliation table |
| Fuzzy high-confidence suggestion | `bg-amber-50 border-amber-200` row highlight | Stage 2 score >= 0.85 |
| Fuzzy low-confidence suggestion | `bg-muted` row highlight | Stage 2 score 0.5вЂ“0.84 |
| Unmatched line | no highlight | Stage 2/3 вЂ” user action required |

Currency formatting:
- Positive amounts (credit): `text-green-700` + `formatBRL(amount)` (mirrors CashFlowTotals `entradas`)
- Negative amounts (debit): `text-red-600` + `в€’${formatBRL(Math.abs(amount))}` (U+2212 true minus, NOT hyphen)
- Neutral/net: `text-foreground` or conditional red if negative (mirrors CashFlowTotals `saldo`)
- Zero: `R$ 0,00` вЂ” never blank
- All amounts via `formatBRL()` from `src/lib/format/money.ts`

## Interaction Contracts

### Read-Only Gate (D-23)
- Roles `auditor`, `dpo`, `sГіcio`: render tables and detail views; hide/disable all write actions (AГ§Гµes column hidden or shows only "Ver Detalhes").
- Roles `receptionist`, `dentist`: no access to `/clinica/financeiro/contas-a-pagar`, `/conciliacao`, `/repasse`, `/rpa` вЂ” handled by proxy module permissions (D-25).
- Roles `admin`, `superadmin`, `financeiro`: full write access.
- Use `assertNotReadOnly()` (Phase 7 pattern) in all Server Actions for write operations.

### Estorno / Cancelamento Approval Gate (D-24)
Any estorno, cancelamento de CP pago, or cancelamento de RPA emitido opens an `AlertDialog` that collects a motivo `Textarea` before calling `createApprovalRequest`. The dialog must NOT call the destructive action directly вЂ” it must always route through `approval_requests`. Title must name the specific action ("Cancelar RPA RPA-UNIT-000001").

### URL State (nuqs pattern вЂ” all filters)
All filter selects and month navigation use `useQueryState` from `nuqs`. All list pages wrap in `<NuqsAdapter>` (imported from `nuqs/adapters/next/app`). Filter state is preserved on month navigation (mirrors `monthHref()` function in fluxo-de-caixa/page.tsx).

### Currency Input Pattern (BRL mask)
All monetary input fields use the `handleAmountBlur` / BRL-mask pattern from `BankAccountFormDialog.tsx`:
- Field schema type: `z.string()` named `*Str` (e.g. `valorStr`)
- On blur: format to `R$ X.XXX,XX`
- On submit: `parseFloat(value.replace(/[^\d,]/g, '').replace(',', '.'))` в†’ `NUMERIC(12,2)`
- No Zod `.default()` (D-133 pattern)

### Date Picker Pattern
All date fields use `Popover` + `Calendar` from shadcn/ui, with format display `dd/MM/yyyy` using `date-fns/ptBR` locale. Mirrors `TransactionModal.tsx` `transactionDate` field exactly.

### Conciliation Stage Colors (accessibility)
Stage 1 (exact match вЂ” auto-conciliated): green row highlight with "Conciliado" badge вЂ” these rows are read-only, no action needed.
Stage 2 (fuzzy suggestion): amber row highlight. "Confirmar" (outline button) and "Recusar" (ghost button) are the only visible actions. After confirmation, row turns green.
Stage 3 (N:1): triggered manually; opens NToOneBuilder Sheet.
Unmatched: no highlight; DropdownMenu with "Buscar LanГ§amento", "Criar LanГ§amento", "Ignorar".

### RPA Calculated Preview
The INSS/IRRF/ISS breakdown in `RpaFormDialog` updates on each BRL-masked blur event (not on every keystroke). Calculation runs client-side using pre-loaded tax bracket data passed as props (to avoid round-trips). The preview is clearly labeled "Estimativa вЂ” valores definitivos calculados no servidor ao emitir."

### CompetГЄncia Selector
A reusable `CompetenciaSelector` client component used on both Repasse and RPA screens. Renders as `"text-sm font-semibold capitalize"` label + prev/next chevron buttons (mirrors CashFlowPage month navigation). Uses `useQueryState('competencia')` nuqs URL state. Format displayed: "Jun/2026".

### PDF Download
- RPA PDF and Demonstrativo PDF: always via signed URL (TTL=60s), never expose `pdf_storage_path`.
- Opens in new tab: `window.open(signedUrl, '_blank')`.
- Button label: "Visualizar PDF" (not "Download" вЂ” the URL may render in-browser).
