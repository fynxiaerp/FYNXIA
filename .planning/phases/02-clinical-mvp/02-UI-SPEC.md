## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | p-1 / gap-1 | Icon gaps, badge padding, tooth SVG internal gap |
| sm | 8px | p-2 / gap-2 | Input field inner padding, compact list items |
| md | 16px | p-4 / gap-4 | Default element spacing, card padding, form field gap |
| lg | 24px | p-6 / gap-6 | Section padding, modal content padding |
| xl | 32px | p-8 / gap-8 | Layout gaps between major page sections |
| 2xl | 48px | p-12 / gap-12 | Major section breaks (e.g., between calendar and sidebar) |
| 3xl | 64px | p-16 / gap-16 | Page-level top/bottom padding on public pages |

**Exceptions:**
- Touch targets (mobile): minimum 44px height for all interactive elements on `/agendar/[clinic-slug]` (public page, used on smartphone)
- Odontogram tooth elements: 30px width × 35px height per tooth (fixed SVG geometry from RESEARCH.md Padrão 5) — does not follow 4pt scale internally
- Signature canvas: 600px × 200px (fixed, defined by signature_pad integration requirement)
- FullCalendar: internal spacing controlled by FullCalendar CSS variables; do not override with Tailwind utilities inside `.fc-*` selectors

## Color

All color values are CSS custom properties defined in `globals.css` using oklch. Use semantic tokens — never hardcode oklch values in component code.

| Role | CSS Token | Light Value | Dark Value | Usage |
|------|-----------|-------------|------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) = white | oklch(0.145 0 0) = near-black | Page background, form backgrounds |
| Secondary (30%) | `--card` / `--sidebar` | oklch(1 0 0) / oklch(0.985 0 0) | oklch(0.205 0 0) | Cards, sidebar, nav panel, modals, popover |
| Accent (10%) | `--primary` | oklch(0.205 0 0) = near-black | oklch(0.922 0 0) = near-white | See reserved list below |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) = red | oklch(0.704 0.191 22.216) = lighter red | Destructive actions only |
| Muted | `--muted` / `--muted-foreground` | oklch(0.97 0 0) | oklch(0.269 0 0) | Inactive states, secondary text, disabled inputs |
| Border | `--border` | oklch(0.922 0 0) | oklch(1 0 0 / 10%) | All dividers, input borders |

**Accent reserved for (explicit list):**
1. Primary CTA buttons (e.g., "Salvar Agendamento", "Registrar Prontuário", "Assinar e Enviar")
2. Selected dentist name in the agenda dropdown (active state)
3. Active navigation link in sidebar
4. Focus ring (`--ring`) on keyboard-focused interactive elements

**Accent is NOT used for:** status badges, calendar event backgrounds (see odontogram and calendar status colors below), table row hover, empty state icons.

## Interaction Contracts

### Agenda Calendar Interactions

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| View appointment | Click on event block | Popover appears with status, patient name (masked), time, dentist. "Abrir" link opens modal |
| Create appointment | Click on empty slot / drag-select | Dialog opens with pre-filled start/end time. Dentist pre-set from dropdown. |
| Edit appointment | Click event → "Editar" in popover | Same Dialog, fields pre-filled, "Salvar" runs updateAppointment Server Action |
| Cancel appointment | Click event → "Cancelar" in popover | Inline confirmation in popover: "Confirmar cancelamento? [Cancelar consulta]" |
| Drag-to-reschedule | Drag event to new slot | `eventDrop` fires updateAppointment Server Action. On GIST conflict (23P01): revert event + show Alert "Horário indisponível" |
| Dentist filter | Select dropdown | nuqs URL state `?dentist={id}` updates. Calendar re-filters. Page reload preserves selection. |
| Week navigation | FullCalendar prev/next buttons | Built-in FullCalendar behavior. Week in URL via nuqs `?week={ISO date}` |

### Patient Form Interactions

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| CPF field | On blur | Format mask: `000.000.000-00`. Validate format (Zod). If duplicate in tenant: `Alert` error inline |
| Date of birth | Focus | shadcn Calendar in Popover opens. Select returns formatted DD/MM/YYYY |
| Save patient | "Salvar Paciente" button click | RHF validates → Server Action `createPatient` / `updatePatient`. On success: redirect to patient chart. On error: inline Alert |
| Sensitive fields | No special mask in form | Plaintext entry; AES-256 applied in Server Action before INSERT |

### Odontogram Interactions

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| View tooth detail | Hover on tooth (editable mode) | Tooltip shows tooth number (FDI) + current status label (pt-BR) |
| Edit tooth status | Click on tooth (admin/dentist only) | Dialog opens: "Dente {number}" + Select for status + Textarea for notes. "Salvar" runs updateDentalRecord |
| Read-only mode | When user role is receptionist/patient | `onClick` prop is no-op. Cursor is `default`. Tooltip still shows status. No Dialog. |
| Extracted tooth | Status = `extraido` | SVG renders X cross lines over tooth shape in gray (#6b7280) |

### Anamnesis Canvas Interactions

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Draw signature | Touch/mouse on canvas | signature_pad captures smooth bezier curves. penColor: #111827 |
| Clear canvas | "Limpar" button | `padRef.current.clear()`. "Assinar e Enviar" button returns to disabled state |
| Confirm signature | "Confirmar Assinatura" button | `pad.toDataURL('image/png')` passed to parent. "Assinar e Enviar" becomes enabled |
| Submit anamnesis | "Assinar e Enviar" button | Disabled until signature confirmed. POST to Server Action `submitAnamnesis`. On success: show immutable signed view. Token marked as used. |
| Token expired/used | Page load with invalid token | Full-page error state: "Este link de anamnese expirou ou já foi utilizado. Solicite um novo link à sua clínica." No form rendered. |

### PDF Download Interaction

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Download prontuario | "Baixar PDF" button on patient chart | GET `/api/patients/[id]/prontuario.pdf`. Button shows loading spinner (skeleton) during generation. Browser triggers download dialog. |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Odontogram teeth keyboard navigation | Each `<g>` element must have `role="button"` + `tabIndex={0}` + `onKeyDown` (Enter/Space = onClick) in editable mode |
| Canvas signature (SignatureCanvas) | `aria-label="Área de assinatura"` on `<canvas>`. Adjacent text: "Desenhe sua assinatura no campo acima." |
| Appointment status color-only differentiation | Each calendar event must include `aria-label` with status text (e.g., "Consulta confirmada — João Silva — 09:00-10:00") |
| Form error messages | All RHF errors rendered via `<FormMessage>` (shadcn Form component) — `role="alert"` applied automatically |
| Destructive confirmation dialogs | Focus must land on "Cancel" (safe action) button first. Destructive button is not the default focus. |
| Touch targets (public pages) | Minimum 44×44px on `/agendar` and `/anamnese` — use `min-h-[44px]` + `min-w-[44px]` on all clickable elements |
| Agenda header [+] button (icon-only) | Must have `aria-label="Novo Agendamento"` on the button element. Alternatively, render as labeled button "+ Novo Agendamento" to avoid the icon-only pattern entirely. |
