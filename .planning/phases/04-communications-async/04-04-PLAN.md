## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet -> cron endpoints | Anyone can hit /api/cron/*; must reject without CRON_SECRET |
| cron (service role) -> appointments/patients/receivables | Cross-tenant reads via admin client; each tenant processed in isolation |
| outbox payload -> Meta/Resend | Patient PHI (name/phone/email) flows out to message providers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-cron-E | Elevation of Privilege | reminder-dispatch endpoint | mitigate | Authorization: Bearer ${CRON_SECRET} -> 401 on mismatch before any DB query (mirrors collection-ruler) |
| T-4-cron-S | Spoofing | unauthenticated cron trigger | mitigate | Same CRON_SECRET guard; Vercel injects the secret on scheduled invocation |
| T-4-cron-dup | Tampering | duplicate sends on at-least-once cron delivery | mitigate | message_log UNIQUE(appointment_id,channel,type) + outbox idempotency_key UNIQUE + worker attempts-before-send + drain selects only status='pending' |
| T-4-cron-I | Information Disclosure | cross-tenant data mixing | mitigate | Per-row tenant_id from the appointment (appt.tenant_id, never from ReminderTarget); payload scoped to one patient; logBusinessEvent logs IDs only |
| T-4-cron-runtime | (reliability) | Edge runtime breaks graph.facebook fetch | mitigate | export const runtime = 'nodejs' (Pitfall 7) |
| T-4-d05-link | Information Disclosure | wrong Asaas payment link in WhatsApp | mitigate | Prefer Asaas API-resolved invoiceUrl over assumed URL pattern; verify field name before use |
| T-4-env-token | Information Disclosure | WHATSAPP_* in client bundle | mitigate | .env.local.example documents non-NEXT_PUBLIC vars; verify config gate asserts no NEXT_PUBLIC_WHATSAPP |
