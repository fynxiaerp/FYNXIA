Build the channel + queue infrastructure: a Meta WhatsApp Cloud API client (typed fetch wrapper, no SDK), the MessageQueue abstraction with an OutboxQueue implementation over message_outbox, and the worker drain loop that sends pending rows via the WhatsApp client or Resend.

Purpose: Decouple enqueue callers from the queue impl and the send mechanism from the channel. Plan 04 wires the cron to this.

</objective>

Create `src/lib/messaging/worker.ts`:
- `import 'server-only'`.
- `export async function drainOutbox(admin = createAdminClient(), batchSize = 100): Promise<{ processed: number; failed: number; skipped: number }>`.
- Query: `admin.from('message_outbox').select('*').eq('status','pending').lte('scheduled_for', new Date().toISOString()).order('scheduled_for', { ascending: true }).limit(batchSize)`. After fetch, filter rows where `row.attempts < row.max_attempts` (Supabase JS cannot compare two columns in a filter — do it in JS).
- Loop with a per-row `try { ... } catch { ... }`:
  1. Bump attempts first: `update({ attempts: row.attempts + 1, last_attempted_at: now }).eq('id', row.id)`.
  2. Send by channel: `whatsapp` → `sendTemplateMessage(row.payload as WhatsAppTemplateParams)` (check `result.success`; if false, treat as failure and capture `result.errorCode`); `email` → `getResend().emails.send(row.payload as ...)`.
  3. On success → `update({ status: 'sent', sent_at: now })`; `logBusinessEvent({ tenantId: row.tenant_id, actorId: null, action: 'message.sent', details: { outbox_id: row.id, channel: row.channel } })`; `processed++`.
  4. On failure → compute `permanent = row.channel === 'whatsapp' && isPermanentError(errorCode)`; `final = permanent || row.attempts + 1 >= row.max_attempts`; `update({ status: final ? 'failed' : 'pending', error_message })`; if final → `failed++`.
  5. catch (err) → log to console.error, attempt to mark the row failed/pending defensively, continue loop (NEVER rethrow).
- Return the summary.
- IMPORTANT (Pitfall 7): this is a lib — the `runtime = 'nodejs'` declaration belongs on the cron route (Plan 04), not here.
