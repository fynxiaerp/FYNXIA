<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| cron (no session) → nps_responses | Service-role invite writes |
| public (no auth) → submitNpsPublic | Token is the only credential |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-19 | Spoofing | NPS token guessing/enumeration | mitigate | token UUID gen_random_uuid() (122-bit) + 7-day expiry; patient_id must also match. |
| T-18-20 | Tampering | replay / double-submit of a score | mitigate | Atomic UPDATE WHERE token_used_at IS NULL AND score IS NULL; affected-row check (TOCTOU-safe, mirrors WR-05). |
| T-18-21 | Information Disclosure | leaking detractor status to patient | mitigate | Public form never surfaces classification; generic error on invalid token; classify is internal only. |
| T-18-22 | Elevation of Privilege | non-cron hitting the scan route | mitigate | isCronAuthorized(authorization header) guard; 401 otherwise. |
| T-18-23 | Denial of Service | duplicate invites | mitigate | UNIQUE(appointment_id) + outbox idempotencyKey `nps-invite:{apptId}:{channel}`. |
</threat_model>
