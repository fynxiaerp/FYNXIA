## Spacing Scale

**Inherited from Phase 2 — identical 8-point scale. No new exceptions for Phase 3.**

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | p-1 / gap-1 | Badge padding, amount column gaps |
| sm | 8px | p-2 / gap-2 | Table cell inner padding, compact row spacing |
| md | 16px | p-4 / gap-4 | Card padding, form field gap, page content padding |
| lg | 24px | p-6 / gap-6 | Section padding, modal content padding |
| xl | 32px | p-8 / gap-8 | Layout gaps between major page sections |
| 2xl | 48px | p-12 / gap-12 | Major section breaks (totals block from transaction list) |
| 3xl | 64px | p-16 / gap-16 | Not used in Phase 3 (no public financial pages) |

**Exceptions (Phase 3-specific):**
- Cash flow totals row (entradas / saídas / saldo): min-h-[72px] cards — intentional break from 8pt to give financial totals visual prominence without disrupting the scale.
- Pix QR code display area: fixed 200×200px (standard QR readability constraint — not a spacing token).

## Typography

**Inherited from Phase 2 — same 4 roles, 2 weights. No new type roles.**

| Role | Size | Weight | Line Height | Tailwind Classes | Usage in Phase 3 |
|------|------|--------|-------------|-----------------|-----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm font-normal leading-relaxed` | Transaction list rows, receivables table cells, category labels, description fields |
| Label | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold leading-snug` | Table column headers, form field labels, filter labels |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight` | Page titles ("Fluxo de Caixa", "Contas a Receber", "Cobranças"), modal titles, section headings |
| Display | 28px | 600 (semibold) | 1.2 | `text-2xl font-semibold leading-tight` | Not used in Phase 3 (no public-facing financial pages) |

**Financial amount typography (new rule for Phase 3):**
- Monetary amounts in summary/totals cards: `text-2xl font-semibold leading-tight` (Display role applied to amounts — NOT to labels).
- Monetary amounts in table cells and transaction lists: `text-sm font-semibold leading-relaxed tabular-nums` — `tabular-nums` ensures digit alignment in columns.
- Format: BRL locale. Pattern: `R$ 1.234,56`. Use `toLocaleString('pt-BR', { style: 'currency', currency: 'BRL' })`. Never hardcode `R$` prefix without locale formatting.

**PDF typography (ReceiboPDF — same contract as ProntuarioPDF):**
- Font family: Roboto (same `Font.register()` call — reuse from `ProntuarioPDF.tsx`)
- Body: 11pt, weight 400, lineHeight 1.5
- Section heading: 14pt, weight 700
- Metadata (dates, IDs, provider_charge_id): 9pt, weight 400, color #6B7280
- Amount fields in PDF: 12pt, weight 700, color #111827

## Color

**All tokens inherited from Phase 2 globals.css — no new CSS variables.** Financial-specific semantic colors are Tailwind classes only (not new CSS variables).

| Role | CSS Token | Usage in Phase 3 |
|------|-----------|-----------------|
| Dominant (60%) | `--background` | Page background, form backgrounds, cash flow page background |
| Secondary (30%) | `--card` / `--sidebar` | Totals summary cards, transaction list card, receivables table container, modal content |
| Accent (10%) | `--primary` | Primary CTA buttons only (see reserved list below) |
| Destructive | `--destructive` | Cancel charge action only |
| Muted | `--muted` / `--muted-foreground` | Zero-state amounts, secondary metadata, "vencido" body text, disabled collection ruler toggles |
| Border | `--border` | All table dividers, card borders, QR code container border |

**Accent reserved for (Phase 3 — explicit list):**
1. "Emitir Cobrança" (primary CTA button on charge creation form)
2. "Lançar Transação" (primary CTA button on cash flow manual entry modal)
3. "Baixar Recibo PDF" (primary CTA button on completed payment row)
4. Active navigation link "Financeiro" in sidebar
5. Focus ring (`--ring`) on keyboard-focused interactive elements

**Accent is NOT used for:** status badges (pendente/pago/vencido), amount totals, category chips, table row hover.

### Financial Status Badge Colors (Phase 3-specific)

Applied via shadcn `Badge` component with Tailwind variant overrides. These are the ONLY new semantic color applications in Phase 3.

| Status | Badge Classes | When Applied |
|--------|--------------|--------------|
| pendente | `bg-amber-100 text-amber-800 border-amber-200` | Charge not yet paid, due date in the future |
| pago | `bg-green-100 text-green-800 border-green-200` | Payment confirmed (webhook or manual) |
| vencido | `bg-red-100 text-red-800 border-red-200` | `due_date < today AND status != 'pago'` — derived at read time (D-04), no DB column for this state |

**Implementation rule:** `vencido` is NEVER a stored DB value. The component derives it client-side: `const displayStatus = status !== 'pago' && isPast(dueDate) ? 'vencido' : status`. Use `date-fns` `isPast()` or equivalent.

### Cash Flow Sign Colors

| Direction | Amount Color Class | Label |
|-----------|--------------------|-------|
| Entrada (receita) | `text-green-700` | Positive sign prefix: `+` |
| Saída (despesa) | `text-red-600` | Negative sign prefix: `−` (U+2212 minus, not hyphen) |
| Saldo (total) | `text-foreground` (neutral) if positive; `text-red-600` if negative | No prefix on totals card |

## Interaction Contracts

### Cash Flow Month Navigation

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Change month | Click "← Mês ant." or "Próx. →" | nuqs updates `?month=YYYY-MM`. Server Component re-renders. URL is shareable/bookmarkable. |
| Default month | Page load (no `?month`) | Defaults to current month (server-side `new Date().toISOString().slice(0, 7)`). |
| Add transaction | Click "+ Lançamento" | Dialog opens (TransactionModal). On submit: Server Action `createTransaction`. On success: Dialog closes + page router.refresh(). On error: inline Alert in Dialog. |
| Filter by type | Select "Todos / Receitas / Despesas" | nuqs `?tipo=entrada|saida`. Client-side filter on fetched data — no additional fetch. |
| Filter by category | Select from category list | nuqs `?category={id}`. Same behavior as tipo filter. |

### Receivables Table Interactions

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Expand installments | Click accordion row for installment charge | shadcn Accordion expands. Shows each parcel with due date, amount, status badge. No additional fetch (parcels loaded with parent). |
| Filter by status | nuqs `?status=pendente|pago|vencido` via filter Select | TanStack Table columnFilters applied. URL shareable. |
| Download receipt | "Recibo" button on pago row | GET `/api/financeiro/charges/[id]/recibo.pdf`. Button shows Skeleton loading state during generation. Browser triggers download. |
| View charge details | Click row | Sheet (drawer) opens with full charge detail: patient, amount, parcel breakdown, Asaas charge ID, payment timestamps. |

### Charge Creation Interactions

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Patient lookup | Focus on Paciente Select/search | Popover with patient search by name or CPF (tenant-scoped). Select returns patient `id`. |
| Toggle parcelamento | Switch "Parcelado?" | RHF `setValue` triggers conditional field render (useWatch on switch field). |
| Valor field mask | Input focus/blur | Format on blur: `1234.56` → `R$ 1.234,56`. Strip non-digits on input. Store raw numeric value in RHF state (not the formatted string). |
| Submit charge | "Emitir Cobrança" button | Disabled during submission (loading spinner). Server Action `createCharge`. On success: result block renders below form (no page navigation). On Asaas API error: Alert with error message and Asaas error code. |
| Copy Pix code | "Copiar código Pix" button | `navigator.clipboard.writeText(pixCode)`. Button label changes to "Copiado!" for 2 seconds (setTimeout). Returns to original label. |
| Download receipt (after charge) | "Baixar Recibo PDF" | GET `/api/financeiro/charges/[id]/recibo.pdf`. Same pattern as prontuario PDF download. |

### Collection Ruler Interactions

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Toggle due-date reminder | Switch click | RHF controlled switch. "Salvar Configurações" must be clicked to persist. No auto-save. |
| Toggle overdue reminder | Switch click | Same as above. If toggled on: "Intervalo (dias)" Input becomes enabled (RHF `disabled` prop controlled by `useWatch` on the switch). |
| Save config | "Salvar Configurações" button | Server Action `saveCollectionRuler`. On success: toast or inline success state. On error: inline Alert. |

### PDF Receipt Download

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Download | "Baixar Recibo PDF" button | GET `/api/financeiro/charges/[id]/recibo.pdf`. Route uses `export const runtime = 'nodejs'`. On success: browser download dialog. On error: Alert "Não foi possível gerar o recibo. Tente novamente." |

## Money Formatting Contract

All monetary values in the UI must follow Brazilian Real formatting. This is a Phase 3-specific rule with no Phase 2 equivalent.

| Rule | Value |
|------|-------|
| Format function | `amount.toLocaleString('pt-BR', { style: 'currency', currency: 'BRL' })` |
| Output format | `R$ 1.234,56` (period as thousands separator, comma as decimal) |
| Negative amounts | `−R$ 1.234,56` (U+2212 true minus, applied in component — not from locale) |
| Zero value | `R$ 0,00` (never blank) |
| Input field | type="text" with mask on blur. Raw numeric value stored in RHF state. |
| Table column alignment | `text-right tabular-nums` on all monetary cells |
| PDF amounts | `amount.toLocaleString('pt-BR', { style: 'currency', currency: 'BRL' })` — same function, applied server-side in ReceiboPDF |
