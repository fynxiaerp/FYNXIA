<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Asaas → webhook endpoint | Untrusted external POST processed under service role (no session) |
| client → createCharge Server Action | Authenticated staff input crossing into the payment gateway |
| Vercel function → Asaas REST | Outbound API call carrying the secret API key |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-webhook-S | Spoofing | /api/webhooks/asaas | mitigate | Validate `asaas-access-token` header against `ASAAS_WEBHOOK_SECRET` BEFORE any processing; 401 on mismatch |
| T-3-webhook-T | Tampering | duplicate / replayed event | mitigate | UNIQUE `webhook_events.asaas_event_id` upsert dedup + pre-insert check on `(receivable_id, type='receita')` — no double credit |
| T-3-webhook-I | Information Disclosure | amount/status from payload trusted blindly | mitigate | Update local receivable by `provider_charge_id`; record only event.payment.value/status; never derive tenant from payload — derive via the matched charge's tenant_id |
| T-3-charge-E | Elevation of Privilege | createCharge | mitigate | getActor role gate (admin/dentist/receptionist/superadmin); RLS WITH CHECK on charges/receivables |
| T-3-charge-I | Information Disclosure | service role key | mitigate | createAdminClient is server-only (`import 'server-only'`); never NEXT_PUBLIC_ |
| T-3-card | Info Disclosure / PCI | card data | mitigate | billingType=CREDIT_CARD only; NO raw card numbers stored — Asaas tokenization/checkout only |
| T-3-xss | XSS | Asaas error surfaced to UI | mitigate | Surface only Asaas `description` string; never innerHTML/eval |
</threat_model>
