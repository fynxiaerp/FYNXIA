| Server Action ← direct POST | bypasses middleware redirect; must self-gate |
| Storage bucket icp-certificates | service-role-only; never public URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-08 | Elevation of Privilege | read-only role mutating via direct Server Action POST | mitigate | middleware sets `x-read-only` (now covers financeiro/bi via most-specific route resolution); mutation actions call `assertNotReadOnly()` (Plan 01); role re-checked server-side, never trusted from header alone |
| T-07-09 | Information Disclosure | .pfx file or cert password exfiltration | mitigate | bytes in private bucket (public=false), only service role reads; password stored AES-256-GCM (`cert_password_enc`); only metadata columns are SELECT-able and tenant-scoped |
| T-07-10 | Spoofing | client claiming elevated role via forged x-user-role | mitigate | proxy re-reads role from `public.users` each request (existing pattern); header is derived server-side, not accepted from client |
| T-07-11 | Tampering | malicious cert subject (CNPJ) injected via crafted .pfx | mitigate | extracted metadata treated as display-only data; never executed/interpolated into SQL (parameterized inserts in Plan 06) |
| T-07-12 | Elevation of Privilege | non-admin writing ai_agent_config to raise autonomy | mitigate | ai_agent_config write RLS restricted to admin/superadmin + tenant scope |
