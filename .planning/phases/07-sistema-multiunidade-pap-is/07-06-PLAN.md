<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client upload → Server Action | .pfx + password are untrusted; processed server-side only |
| Server Action → private Storage | only service role touches icp-certificates; bytes never reach client |
| extracted cert metadata | attacker-controlled certificate subject |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-18 | Information Disclosure | .pfx bytes or password leaking to client | mitigate | bytes uploaded via service-role admin client to public=false bucket; password AES-256 encrypted before insert; getCertificate return type `Omit<...'cert_password_enc'|'storage_path'>` — compile-time guarantee secrets never reach client |
| T-07-19 | Elevation of Privilege | read-only/non-admin uploading cert or raising AI autonomy | mitigate | `assertNotReadOnly()` + admin/superadmin(/ti for cert) gate in both actions; RLS write policy (Plan 03) backstop |
| T-07-20 | Denial of Service | oversized .pfx upload | mitigate | zod 5 MB size cap rejected before forwarding to Storage |
| T-07-21 | Tampering | malicious certificate subject (CNPJ) used downstream | mitigate | extracted metadata is display-only, parameterized inserts; expired certs rejected (`not_after < now()`) |
| T-07-22 | Spoofing | invalid cert / wrong password accepted | mitigate | extractPfxMetadata throws on bad password/parse → upload aborted with friendly error |
</threat_model>
