## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| authenticated client → new tables | Any tenant session can attempt reads/writes; RLS is enforcement |
| cron/agent (service role) → bi_alerts | Service role bypasses RLS; bi_alerts must have NO authenticated write policy |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-04 | Information Disclosure | partner_shares cross-sócio read | mitigate | SELECT policy: admin/superadmin OR (socio AND user_id = auth.uid()) |
| T-19-05 | Elevation of Privilege | socio writing partner_shares | mitigate | write policy admin/superadmin only (A1); socio has NO INSERT/UPDATE grant |
| T-19-06 | Tampering | client writing bi_alerts to fake forecasts | mitigate | NO authenticated INSERT/UPDATE policy; writes only via service role |
| T-19-07 | Information Disclosure | cross-tenant leak on any new table | mitigate | USING (clinic_id = get_my_tenant_id()) + WITH CHECK on every write policy |
</threat_model>
