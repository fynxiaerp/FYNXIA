## Typography

Reused as-is from the established, already-shipped project scale (verified in `PageHeader.tsx`, `KpiCard`, `NpsScoreCard.tsx`, prototype pages) — **do not invent a new scale for this phase.**

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 — table cells, form labels, card descriptions, copy text |
| Label | 12px (`text-xs`) | 400 (regular) | 1.5 — muted meta text, table sub-labels (e.g. "6 dentistas", period captions), badge text |
| Heading | 20px (`text-xl`) | 600 (semibold) | 1.2 — `<h1>` page title (`PageHeader`), section/card titles (`ChartCard`/table header `<h3>`) — always `.font-display` |
| Display | 24px (`text-2xl`) | 600 (semibold) | 1.2 — KPI numeric values (`KpiCard` value, DRE Resultado/Margem, per-sócio R$ value), always `tabular-nums`, always `.font-display` |

Exactly 2 weights declared: **400 (regular)** and **600 (semibold)**. Do not introduce `font-medium` (500) or `font-bold` (700) anywhere in this phase's new components — existing codebase (`KpiCard`, `PageHeader`, table headers) uses only 400/600, and this phase must match.

## Color

Fully token-based (dual light/clinical + dark/brand theme, already wired in `globals.css`) — **no new hex colors or new CSS custom properties are introduced by this phase.**

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` / `--card` (light: `hsl(0 0% 100%)` / `hsl(0 0% 98%)`; dark: `hsl(240 20% 6%)` / `hsl(240 20% 10%)`) | Page background, all card surfaces (KPI cards, chart cards, table containers) across DRE/Orçamento/Societário/BI |
| Secondary (30%) | `--secondary` / `--muted` / `--sidebar` | Table header rows, progress-bar track (occupancy bar `bg-muted`), sidebar nav for the 4 new menu entries, tab list background |
| Accent (10%) | `--primary` (light: teal `hsl(185 100% 26%)`; dark: cyan `hsl(185 100% 50%)`) | **Reserved exclusively for:** primary CTA buttons (Exportar PDF, Salvar metas, Nova vigência, Copiar do ano anterior), active nav/tab indicator, positive KPI delta icon+text (`DeltaBadge` up-state), occupancy/progress bar fill, DRE positive Resultado/Margem emphasis, chart line/bar/donut segment tones (`chart-1`..`chart-5`, `primary` — used only inside chart primitives, never as a decorative page accent) |
| Destructive | `--destructive` (`hsl(0 84% 60%)`, both themes) | Reserved exclusively for: semaphore "vermelho" (budget deviation >15%, D-15), negative per-sócio prejuízo values (D-27), negative KPI delta (`DeltaBadge` down-state — already `text-destructive` in the existing component), destructive confirmation dialogs ("Encerrar vigência"), error state banners/toasts |

**Additional semantic status colors (ad-hoc Tailwind, not new CSS variables)** — required for the 3-state budget/KPI semaphore (D-15/D-33) that this phase introduces and that no existing shadcn token covers:

| Status | Class (light / dark) | Usage — reserved for |
|--------|----------------------|----------------------|
| Verde (dentro da meta) | `text-green-700` / `dark:text-green-400` | Budget deviation <5% (D-15), KPI at/above meta, positive NPS score — **reuses the exact pattern already established in `NpsScoreCard.tsx`** (not a new convention) |
| Amarelo (atenção) | `text-amber-600` / `dark:text-amber-400` | Budget deviation 5–15% (D-15) — **first use of amber in this codebase**; confirm with checker this doesn't collide with any existing warning convention before implementation |
| Vermelho (crítico) | `--destructive` token (not ad-hoc red) | Budget deviation >15%, prejuízo — reuses the existing destructive token, not a new red |

Do not use amber/green/red as background fills for large surfaces — text/icon/badge-border only, matching the existing `NpsScoreCard` restraint (color carries meaning via text + tabular-nums, not full-card tinting).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (generic) | Verb + noun, never bare "OK"/"Salvar" alone — see per-screen table below |
| Empty state heading | Specific to data-missing condition — see per-screen table below |
| Empty state body | Copy + concrete next step (never just "sem dados") |
| Error state | Problem description + explicit retry/next-step path |
| Destructive confirmation | Action name + irreversibility statement + Confirm/Cancel |

| Screen | Primary CTA | Empty State | Error State | Destructive Action |

## Phase-Specific Component Notes (beyond template — required for D-37 compliance)

### New chart/KPI component namespace (D-37)

`src/components/prototipos/charts.tsx` (`KpiCard`, `ChartCard`, `LineChart`, `BarChart`, `DonutChart`) is **reference-only** for this phase — do not import from `@/components/prototipos/*` in any of the 4 new screens, even though existing production code (`convenios`/`nfse` pages) already does this today. Build a new, non-`prototipos` namespace, e.g. `src/components/bi/charts.tsx` or `src/components/relatorios/charts.tsx`.

**Recommended approach:** copy the exact same dependency-free pure-SVG/CSS technique (proven, zero bundle cost, themeable via `chart-1`..`chart-5`/`primary` tokens) — do not introduce `recharts` or any charting library (none installed, none needed per RESEARCH.md). The new components must be pixel/behavior-equivalent to the prototype ones (same KPI card layout, same line/bar/donut rendering approach) — only the import path and namespace change.

**New primitive needed beyond the prototype set:** a horizontal progress/occupancy bar (used for DRE unit-ranking "ocupação" column, D-04, mirroring `dashboard-franquias` prototype's inline `<div className="h-1.5 w-20 rounded-full bg-muted overflow-hidden"><div className="h-full rounded-full bg-primary" style={{width:...}}/></div>` pattern). Recommend extracting this into the new chart namespace as a small `OccupancyBar` component rather than inlining it again — same token usage (`bg-muted` track, `bg-primary` fill), not a new shadcn `Progress` install.

### Screen-level layout contract (applies to all 4 screens)

- `PageHeader` (existing, reused as-is) with `title`, `breadcrumbs` (e.g. `[{label:'Relatórios'}]` — top-level module, no "Protótipos" crumb since these are production screens), and `actions` slot for the period/unit selector + "Exportar PDF" button.
- Content wrapper: `<div className="p-6 max-w-6xl mx-auto w-full space-y-6">` — matches existing prototype convention exactly, do not widen beyond `max-w-6xl`.
- KPI row: `grid grid-cols-2 lg:grid-cols-4 gap-4` (DRE/Orçamento) or `lg:grid-cols-5` (BI, matching `dashboard-franquias`'s 5-KPI row) — pick per screen's actual KPI count, never fewer than 2 columns on mobile.
- Table sections: `rounded-xl bg-card ring-1 ring-foreground/10 overflow-hidden` wrapper with a `px-4 py-3 border-b border-border` header block (`<h3 className="font-display text-base font-semibold">` + `<p className="text-sm text-muted-foreground">`) — **note:** this header uses `text-base` (16px) not the Heading role's 20px; this is the established sub-section-header size distinct from the page-level `<h1>`. Treat 16px/600 as a permitted 5th typographic instance reserved exclusively for in-card section headers (`ChartCard`/table-wrapper titles) — do not use it elsewhere.

### DRE-specific (D-04, D-05, D-06, D-08)

- Consolidated view ("Todas as unidades"): DRE summary block + unit-ranking table below it, same visual pattern as `dashboard-franquias` prototype's ranking table (position badge, name+city/dentists sub-line, revenue, occupancy bar, delinquency badge, delta badge) — reuse this exact column set, adapted to DRE terms (Receita/Despesa/Resultado/Margem instead of raw revenue).
- Drill-down (D-05) and cost-center expansion (D-06): use `Accordion` (already installed, same primitive as the Plano de Contas tree from Phase 14) for expand/collapse per line — do not build a custom disclosure widget.
- % of revenue column (D-08): render as `text-xs text-muted-foreground tabular-nums` inline next to each line's R$ value, e.g. "R$ 12.400 · 18% da receita" — Label role (12px/400).

### Orçamento-specific (D-15 semaphore, D-18 lock)

- 12-month editable grid: each month cell shows Meta (editable `Input`, disabled/grayed when locked per D-18) + Realizado (read-only) + semaphore-colored deviation %. Locked past-month cells: `disabled` input styled with `bg-muted text-muted-foreground` (matches existing disabled-input convention in the codebase, e.g. `CostCenterFormDialog`'s immutable `unitId` field) plus a small lock icon (`lucide-react` `Lock`, size 3.5) for discoverability — do not silently disable without a visual cue.

### Societário-specific (D-23, D-24, D-27)

- Each sócio row: name + percentual (badge, `variant="secondary"`) + R$ calculado (Display role, 24px/600, `tabular-nums`) — **negative values (D-27) render in `--destructive` color with a leading minus sign, never hidden or zeroed.**
- Vigência history: expandable/table view showing past vigências (start/end dates) — read-only list, not editable inline (editing = creating a new vigência via "Nova vigência", never mutating a closed one).
- Sócio's own-row-only view (D-24): render as a single `Card` (not a table) when the viewer's role is `socio` — no table chrome implying "there are more rows you can't see."

### BI-specific (D-38 tabs + fixed alert section)

- Layout: `Tabs` (shadcn, already installed) for "Operacional | Profissionais | CRC | Estoque/TISS" — the "Alertas & Previsões" section renders **outside and above** the `Tabs` component tree (a fixed `section` before `<Tabs>`), not as a 5th tab, so it stays visible across all tab switches per D-38.
- Alert cards: reuse `ChartCard`-style wrapper (new namespace) with severity-colored left border or leading icon (verde/amarelo/vermelho semaphore, or a neutral `Info` icon for D-32 "insufficient data" notices) + LLM narrative text body (Body role, 14px/400) + optional "Revisar sugestão" link only when the alert has a concrete `approval_requests` entry (D-35 — informational-only alerts get no action link).
