## Spacing Scale

Inherited from Phase 6 (project standard). Reproduced for executor reference.

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, badge padding, indent marker gaps |
| sm | 8px | `p-2` / `gap-2` | Table cell padding, compact row spacing |
| md | 16px | `p-4` / `gap-4` | Default card padding, form field gap |
| lg | 24px | `p-6` / `gap-6` | Section padding, page content padding |
| xl | 32px | `p-8` / `gap-8` | Layout gaps between sections |
| 2xl | 48px | `p-12` / `gap-12` | Major section breaks |
| 3xl | 64px | `p-16` / `gap-16` | Page-level spacing (public pages only) |

**Phase 14 exceptions:**

- Chart-of-accounts tree: indent per level = `pl-6` (24px per depth level). Level 0 (grupos): `pl-0`. Level 1 (subgrupos): `pl-6`. Level 2 (contas folha): `pl-12`.
- Tree row height: `h-10` (40px) — mirrors sidebar nav item height, touch-friendly.
- Account code column: fixed `w-24` to align numeric codes (e.g., `1.1`, `1.1.1`) in a monospaced column. Use `font-mono tabular-nums text-sm`.
- Page content max-width: `max-w-5xl mx-auto w-full` — project standard for financial pages.

## Typography

Inherited from Phase 6. Exactly 4 roles, exactly 2 weights (400 + 600).

| Role | Size | Weight | Line Height | Tailwind Classes | Usage |
|------|------|--------|-------------|------------------|-------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm font-normal` | Table cells, form descriptions, tree node labels |
| Label | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold` | Input labels, column headers, section labels, account codes |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight font-display` | PageHeader title, card headings, modal titles |
| Display | 28px | 600 (semibold) | 1.2 | `text-2xl font-semibold leading-tight font-display` | Not used in this phase (no auth/hero surfaces) |

**Phase 14 additions:**

- Account code: `font-mono tabular-nums text-sm font-semibold text-muted-foreground` — Brazilian numeric hierarchy (e.g., `1`, `1.1`, `1.1.1`).
- Tree group node label (non-leaf): `text-sm font-semibold text-foreground` — heavier than leaf to signal grouping.
- Tree leaf node label (conta folha): `text-sm font-normal text-foreground`.
- Monetary value in bank accounts: `text-sm font-semibold tabular-nums` — inherits TransactionList pattern.
- Metadata subtitles (unit affiliation under CC name): `text-xs text-muted-foreground` — pre-approved narrow pattern from Phase 6. **`text-xs` (12px) is restricted to this single use case only** — unit affiliation subtitles under Cost Center name. Do not apply `text-xs` to any other element in this phase; use `text-sm text-muted-foreground` for all other secondary text to avoid a 12/14px gap that creates visual noise.

## Color

Inherited from Phase 6 dual-theme contract. Tokens already in `globals.css`. No new tokens needed for Phase 14.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` / `--background` | `hsl(240 20% 6%)` | Page surfaces, form backgrounds |
| Secondary (30%) | `hsl(0 0% 98%)` / `--card` | `hsl(240 20% 10%)` | Table containers, cards, modals, drawer panels |
| Accent (10%) | `hsl(185 100% 26%)` / `--primary` (light) | `hsl(185 100% 50%)` / `--primary` (dark) | See reserved list below |
| Destructive | `hsl(0 84% 60%)` / `--destructive` | Same | Delete account, delete CC, delete bank account |

**Accent (`--primary` / cyan) reserved for — Phase 14 specific elements:**

1. "Nova Conta" / "Novo Centro de Custo" / "Nova Conta Corrente" primary CTA buttons
2. Focus rings on keyboard-focused tree rows and form inputs (`--ring`)
3. Active sort column header indicators (arrow icon, inherits project standard)
4. Save submit buttons inside Dialog forms ("Salvar Conta", "Salvar Centro de Custo", "Salvar Conta Corrente")

**Accent is NOT used for:** tree expand/collapse chevrons (use `text-muted-foreground`), account type badges, the account code prefix, tree indent lines, table hover rows, empty state icons.

**Phase 14 financial semantic colors (Tailwind classes, not tokens):**

| Type | Classes |
|------|---------|
| Receita / grupo receitas | `text-green-700` (light) / `text-green-400` (dark) — account type badge only |
| Despesa / grupo despesas | `text-red-600` (light) / `text-red-400` (dark) — account type badge only |
| Grupo neutro | `text-muted-foreground` — for root groups with mixed children |
| Conta folha ativa | `text-foreground` |
| Conta folha inativa | `text-muted-foreground line-through` — visually deprioritized without deletion |

**Receita/Despesa badges** (on tree nodes): `Badge` with `variant="outline"` + color class override. Do not use `variant="destructive"` for despesa — that is reserved for destructive actions only.

## Component Inventory

### Tree Component — Chart of Accounts (FCAD-01)

**Decision (CONTEXT.md D-02, Claude's Discretion):** Use `Accordion` (`@base-ui/react/accordion`) as the tree primitive for `chart_of_accounts`. This is already installed (`src/components/ui/accordion.tsx`) and is the only existing collapsible hierarchical component in the codebase.

**Implementation contract:**

```
ChartOfAccountsTree ('use client')
  Accordion (multiple, all open by default)
    AccordionItem [per grupo/subgrupo]
      AccordionTrigger
        [indent spacer: pl-{depth*6}]
        [ChevronDown/Up — text-muted-foreground size-4]
        [account-code — font-mono tabular-nums text-sm font-semibold text-muted-foreground w-20]
        [account-name — text-sm font-semibold (group) / font-normal (leaf)]
        [type Badge — outline variant, receita green / despesa red / grupo muted]
        [actions — Edit + (Add child if group) buttons, ml-auto, show on row hover]
      AccordionContent
        [recursive child rows — indented pl-6 per level]
        [leaf rows rendered as plain div (no AccordionItem) with same row anatomy]
```

**Interaction rules:**

- Grupos (type='grupo') and parent nodes: render as `AccordionItem` (expandable/collapsible).
- Contas folha (leaf nodes, no children): render as a plain `div` with `h-10 flex items-center` — no expand/collapse affordance.
- Default state: all `AccordionItem` nodes open (`defaultValue` = all item IDs).
- Expand/collapse: click on trigger row. Chevron rotates. Children animate with `data-open:animate-accordion-down`.
- Row hover: `hover:bg-muted/50` — reveals the Edit icon button and (for groups) the "+ Conta Filha" icon button.
- Edit action: opens `AccountFormDialog` (Dialog). Account code is auto-computed from parent + next sequence number; editable by admin.
- "+ Conta Filha" action (group rows only): opens `AccountFormDialog` pre-filled with parent context.
- "Nova Conta" button (page-level action): opens `AccountFormDialog` with no parent.
- Cannot delete a conta that has `financial_transactions` referencing it — show `AlertDescription` inline. Can only deactivate (toggle).
- Inactive accounts: rendered with `text-muted-foreground` + `line-through` on name. Still visible in tree (admin needs to see them). Hidden from transaction classification dropdowns.

**Accessibility:**

- `AccordionTrigger` has `aria-expanded` (Radix default). Add `aria-label={`Expandir ${account.code} — ${account.name}`}` on each trigger.
- Leaf rows: `role="listitem"`.
- Edit button: `aria-label={`Editar conta ${account.code}`}`.

## Interaction Contracts

### Chart of Accounts Tree

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Expand group | Click trigger row of Accordion item | Chevron rotates; children animate in (`data-open:animate-accordion-down`) |
| Collapse group | Click trigger row again | Children animate out |
| Hover row | Mouse enter | `bg-muted/50`; action icon buttons appear (opacity-0 → opacity-100, `transition-opacity duration-150`) |
| Edit account | Click pencil icon (hover) | Opens `AccountFormDialog` pre-filled |
| Add child | Click "+" icon on group row (hover) | Opens `AccountFormDialog` with `parent_id` pre-set; `codigo` auto-computed |
| Create root | Click "Nova Conta" in PageHeader | Opens `AccountFormDialog` with no parent |
| Toggle ativo | Toggle Switch in edit dialog | Submits `updateAccount`; tree refreshes; inactive leaf rendered with strikethrough |
| Delete blocked | Attempt to delete referenced account | Dialog shows inline `Alert variant="destructive"`: "Não é possível excluir — conta possui lançamentos vinculados. Desative a conta para ocultá-la dos seletores." |

### Cost Center and Bank Account Tables

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Create | Click primary CTA | Opens Dialog form |
| Edit | Click Edit button in row | Opens Dialog pre-filled |
| Toggle ativo (CC) | Click Switch in row | Immediate `useTransition` update; row badge updates |
| Delete | Not exposed in UI | CCs and bank accounts are deactivated, not deleted |

### Transaction Classification (FCAD-02)

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Pick Categoria | Select dropdown | Auto-populates Conta Contábil from `categoria.account_id` if mapped; shows helper text |
| Override Conta | Select different leaf account | Clears auto-set helper text; user selection takes precedence |
| Pick Centro de Custo | Select dropdown | Default pre-selected (user's unit default CC); overridable |
| Submit without classification | Click "Lançar Transação" | `FormMessage` appears on Conta Contábil and/or Centro de Custo fields: "Campo obrigatório" |
| Auto-posted webhook | System (no user) | Resolver in Server Action: `account_id` from `categoria.account_id`, `cost_center_id` from unit default. No UI blocking. |

### Cancel Button Rationale

"Cancelar" appears in all dialog footers as a secondary dismiss/close action — it closes the dialog without submitting the form and discards any unsaved input. It is not a primary CTA. It does not trigger any mutation or side effect. The label is standard pt-BR for dismiss/close in modal dialogs and is consistent with the rest of the application. No per-dialog variant is needed.

### Destructive Actions

| Action | Confirmation Pattern | Button Label |
|--------|---------------------|--------------|
| Desativar conta contábil | Inline `Alert` on dialog: "Contas inativas ficam ocultas nos seletores de lançamento, mas lançamentos existentes são preservados." + Switch toggle (no separate confirm dialog) | Switch label: "Conta ativa" |
| Desativar centro de custo | Same pattern — inline Alert on dialog + Switch | Switch label: "Centro de custo ativo" |
| (No hard-delete exposed) | N/A | N/A |

No destructive Dialog confirmation is needed in this phase — the most destructive action is deactivation (reversible). If a delete were ever exposed, it would follow the standard Phase 6 Dialog confirmation pattern.

## Accessibility Contract
