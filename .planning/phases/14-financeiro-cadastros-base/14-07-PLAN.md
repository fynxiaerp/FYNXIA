## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-19 | T (Tampering) | manual classification ids cross-tenant | mitigate | Select options come only from RLS-filtered listings; createTransaction (Plan 04) sets tenant_id from actor and the account/CC FKs resolve only within tenant; RLS WITH CHECK is the backstop |
| T-14-20 | I (Info Disclosure) | unit/cc filter leaking other tenant rows | mitigate | listTransactions runs under RLS; the `.eq('cost_center_id', cc)` / `.in(...)` only further narrows already-tenant-scoped rows — a foreign cc id simply matches 0 rows |
| T-14-21 | T (Tampering) | updateCategoryAccount cross-tenant/leaf abuse | mitigate | Plan 04 action validates the account is same-tenant + leaf + matching type before UPDATE |
