## Dual-Theme Contract

Phase 6 introduces two complete token sets. The **light clinical theme** is the default (`<html>` has no class). The **dark brand theme** is activated by adding the `.dark` class to `<html>` (persisted via `next-themes` with `storageKey="fynxia-theme"`, `defaultTheme="light"`, `attribute="class"`).

### Theme Mechanism

```
next-themes ThemeProvider (src/components/providers/ThemeProvider.tsx)
  attribute="class"
  defaultTheme="light"
  enableSystem={false}
  storageKey="fynxia-theme"

Mounted in src/app/layout.tsx wrapping <body>
```

`next-themes` reads the cookie/localStorage before hydration. With Tailwind v4 and `@custom-variant dark (&:is(.dark *))`, no flash occurs on first paint when SSR renders the correct class. The `suppressHydrationWarning` prop is required on `<html>`.

### FYNXIA Brand Gradient

The brand gradient `cyan → magenta` (`hsl(185 100% 50%) → hsl(300 100% 60%)`) and the cyan → purple variant are **brand moments only**. Usage is strictly limited to:
- Sidebar header area (logo background chip)
- Login / signup / forgot-password page backgrounds
- Primary CTA buttons (gradient fill variant — see Button contract)
- Public pages hero heading (gradient text effect)

**Never apply gradient or glow to:** table rows, form inputs, card content areas, data cells, status badges, or any dense-data surface.

## Color Contract — 60/30/10 + Accent Reserved List

### Both Themes

**60% Dominant:** `--background` — page surfaces, form backgrounds, page scroll area behind the sidebar.

**30% Secondary:** `--card` / `--sidebar` — all card containers, the sidebar panel, modals, popovers, sheet drawers, data table containers.

**10% Accent (--primary / cyan):**

Accent color is reserved exclusively for:
1. Primary CTA buttons ("Salvar Agendamento", "Emitir Cobrança", "Novo Paciente", etc.)
2. Active navigation item in the sidebar (background fill + icon + label)
3. Focus rings (`--ring`) on keyboard-focused interactive elements
4. The copilot trigger button background
5. "Perguntar" send button in the copilot sidebar (when input is non-empty)
6. Sorting arrows in active sort column headers
7. Progress indicator fill on the public anamnesis page

**Accent is NOT used for:** decorative nav icons in their default (inactive) state, status badges, empty state icons, table row hover, card hover borders, financial amount text.

**Magenta / Purple (dark theme only — brand moments):**
- `--secondary` (magenta): gradient endpoints on sidebar header chip, login CTA gradient, hover on brand hero elements.
- `--accent` (purple): gradient midpoint only. Never as a standalone fill.

**Destructive (`--destructive`):** Cancel appointment, anonymize patient, cancel charge. Only for irreversible or explicitly destructive actions.

### Type Scale

Exactly 4 roles, exactly 2 weights in application UI (400 + 600). The brand exception for `700` on the FYNXIA wordmark in auth pages is documented below.

| Role | Size | Weight | Line Height | Tailwind Classes | Usage |
|---|---|---|---|---|---|
| Body | 14px | 400 (regular) | 1.5 | `text-sm font-normal leading-relaxed` | Form labels, table cells, sidebar nav labels, message bubbles, descriptions |
| Label | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold leading-snug` | Input labels, column headers, section labels, active sidebar items |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight font-display` | PageHeader title, modal titles, card headings |
| Display | 28px | 600 (semibold) | 1.2 | `text-2xl font-semibold leading-tight font-display` | Auth page wordmark (non-FYNXIA heading usage), public page hero title |

**Weights in application UI: exactly 400 and 600. No `font-medium` (500), no `font-bold` (700) outside the documented brand exception.**

**Brand exception (FYNXIA wordmark only):**
The word "Fynxia" in the sidebar logo area and on auth pages uses `font-bold` (700) via the Space Grotesk `700` weight. This is intentional — it creates the brand wordmark weight. Document it as: `text-2xl font-bold font-display tracking-tight`. This exception applies to exactly one element per screen and is not extended to any other heading.

**Financial amounts:** `text-2xl font-semibold tabular-nums` for totals cards. `text-sm font-semibold tabular-nums text-right` for table cells. Inherited from Phase 3.

**Eliminated violations from audit:**
- Replace all `font-bold` headings (except wordmark exception) with `font-semibold`.
- Replace all `font-medium` usages with `font-semibold` (labels/headers) or `font-normal` (body).
- Remove `text-3xl` usages — max Display size is `text-2xl`. Auth wordmark now uses `text-2xl font-bold font-display` (not `text-3xl`).
- Remove `text-lg` (18px) from equipe page — replace with `text-xl font-semibold` (section heading) or `text-sm font-semibold` (label).
- Remove `text-base` (16px) from empty state headings and card titles — use `text-xl font-semibold` or `text-sm font-semibold`.

**Metadata labels** (column subheadings in cards, prontuario field labels): `text-xs font-semibold uppercase tracking-wide text-muted-foreground` — this narrow pattern is pre-approved.

## Interaction Contracts

### Theme Toggle

| Interaction | Trigger | Behavior |
|---|---|---|
| Toggle theme | Click `ThemeToggle` in SidebarFooter | `setTheme(theme === 'dark' ? 'light' : 'dark')` via next-themes. `<html>` class updates. CSS variables swap. Transition: `transition-colors duration-300` on `<html>`. Preference persisted in localStorage under key `fynxia-theme`. |
| SSR theme | Server render | ThemeProvider reads `storageKey` cookie (next-themes `attribute="class"` + Next.js cookie). Renders correct class on first paint — no flash. |

|---|---|---|

| Mobile drawer | Click hamburger in PageHeader (visible only on `<md`) | Opens Sheet (left, `w-[240px]`, `side="left"`). Sheet contains same AppSidebar content. |
| Persist state | After toggle | Collapse state persisted in Zustand (no localStorage — resets on page load is acceptable; sidebar always opens expanded). |
