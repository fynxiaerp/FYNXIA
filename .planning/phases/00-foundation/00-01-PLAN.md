| Boundary | Description |
|----------|-------------|
| browser ŌåÆ Vercel Edge (middleware) | Untrusted JWT cookie crosses here; must be validated against Auth server |
| build pipeline ŌåÆ client bundle | Server secrets must NOT cross into `.next/static/` |
| application server ŌåÆ external AI/3rd party | Out of scope for Phase 0 (no integrations yet) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-01 | Spoofing (C-4) | src/middleware.ts JWT check | mitigate | Middleware calls `supabase.auth.getUser()` (validates against Auth server), NEVER `getSession()`. Acceptance criteria assert `getUser` present and `getSession` absent. |
| T-00-02 | Information Disclosure (C-2) | src/lib/supabase/admin.ts, src/lib/crypto.ts | mitigate | `import 'server-only'` on every file touching SERVICE_ROLE_KEY / ENCRYPTION_KEY; env vars never NEXT_PUBLIC_; post-build grep `security-check` script. |
| T-00-03 | Tampering (H-4) | src/lib/supabase/middleware.ts cookie handlers | mitigate | `setAll` writes cookies to BOTH request and response, preventing refresh-token race that forces logouts. |
| T-00-04 | Information Disclosure (SEC-08) | src/lib/crypto.ts | mitigate | AES-256-GCM (confidentiality + integrity via auth tag); key never leaves server; 64-char-hex key validation. |
| T-00-05 | Elevation of Privilege | Edge runtime DB access | accept | No DB calls in middleware beyond getUser (Auth server HTTP); API routes pinned to Node runtime; no Edge DB connections introduced this plan. |
</threat_model>
