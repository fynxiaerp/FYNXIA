## Color

All color tokens are CSS custom properties from `globals.css` (Phase 2). No new tokens.

| Role | CSS Token | Phase 5 usage |
|------|-----------|---------------|
| Dominant (60%) | `--background` | Page behind the sidebar overlay |
| Secondary (30%) | `--card` / `--sidebar` | Sidebar panel background (`bg-card`); user message bubble background (`bg-muted`) |
| Accent (10%) | `--primary` | Copilot trigger button icon; "Perguntar" send button fill |
| Destructive | `--destructive` | Not used in the copilot sidebar (no destructive actions; D-05: read-only) |
| Muted | `--muted` / `--muted-foreground` | User message bubble background; suggested prompt chips; empty state icon |
| Border | `--border` | Panel border (left edge), message input border, divider between header and list |

**Message bubble color split:**
- **User message:** `bg-muted` (secondary surface), `text-foreground`, `rounded-xl rounded-br-sm`, right-aligned. Tailwind: `ml-auto max-w-[80%] bg-muted px-3 py-2 rounded-xl rounded-br-sm`.
- **Assistant message:** `bg-card` (same as panel, no fill — appears as plain text with left border marker), `text-foreground`, left-aligned. Tailwind: `mr-auto max-w-[85%] px-3 py-2`. No background fill on assistant messages — keeps visual hierarchy clean and avoids competing surfaces.

**Accent usage in Phase 5 (added to Phase 2 reserved list):**
5. Copilot trigger button **background** — `bg-primary` (the accent fill); the Bot icon (lucide-react) inside is `text-primary-foreground` (contrast color, not accent).
6. "Perguntar" send button when input is non-empty — `bg-primary text-primary-foreground`.

**Accent is NOT used for:** message bubbles, streaming cursor (uses `bg-foreground`), suggested prompt chips, clear-conversation affordance.

## Interaction Contracts

### Copilot Trigger

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Open sidebar | Click CopilotTrigger button | Sheet opens from right. Trigger icon changes to `X`. `aria-expanded="true"`. |
| Close sidebar | Click trigger again or Sheet overlay/close button | Sheet closes. Trigger icon returns to `Bot`. `aria-expanded="false"`. |
| Persistent across pages | Navigate between `/clinica/*` pages | Sidebar state (open/closed) persists in Zustand client store — conversation resets on page navigation (no cross-page memory). |

### Message Submission

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Send message | Click "Perguntar" or press Enter (Shift+Enter = newline) | `handleSubmit()` from `useChat`. Input clears. SuggestedPrompts disappear. Streaming begins. |
| Streaming in progress | After submission | "Perguntar" button disabled. Input disabled. TypingIndicator shown while awaiting first token. Streaming cursor `|` appended to last token during stream. |
| Stream complete | When `isLoading` becomes false | Input re-enabled. Cursor removed. MessageList auto-scrolls to last message. |
| Clear conversation | Click "Limpar conversa" | Calls `useChat`'s `setMessages([])`. No confirmation dialog — no data is mutated (read-only, D-05). SuggestedPrompts re-appear. Input re-focuses. |

### Suggested Prompts (empty state only)

Shown when `messages.length === 0`. Three context-aware chips derived from current `pathname`:

| Pathname | Chip 1 | Chip 2 | Chip 3 |
|----------|--------|--------|--------|
| `/clinica/agenda` | "Quais consultas tenho hoje?" | "Algum horário livre essa semana?" | "Como remarco uma consulta?" |
| `/clinica/financeiro*` | "Qual o total de recebíveis em aberto?" | "Quais pacientes estão inadimplentes?" | "Como gero uma cobrança?" |
| Any other `/clinica/*` | "Como cadastro um paciente?" | "Quais módulos o sistema tem?" | "Como funciona o odontograma?" |

Chip interaction: clicking a chip populates the input field and immediately submits (calls `handleSubmit`). Chips disappear after first message.

**Chip styling:** `inline-flex items-center gap-2 rounded-full border border-border bg-muted px-3 py-2 text-sm text-muted-foreground hover:border-primary hover:text-foreground cursor-pointer transition-colors`. Wrap in a `flex flex-wrap gap-2 px-4 pt-4` container. (Spacing on the 4pt grid: `gap-2`/`py-2` = 8px.)

### Typing Indicator (while awaiting first token)

Shown between user message submission and first streamed token from assistant. Three animated dots:

```
● ● ●  (staggered pulse animation — CSS animate-bounce with delays 0ms / 150ms / 300ms)
```

Tailwind: `flex items-center gap-1 px-3 py-2 mr-auto`. Each dot: `h-2 w-2 rounded-full bg-muted-foreground`.

Remove when first token arrives (i.e., when `messages` array gains the assistant entry with non-empty content).

### Error State

| Error | Display location | Copy |
|-------|-----------------|------|
| AI Gateway unreachable / 5xx | Inline in MessageList as an assistant message with `text-destructive` | "Não foi possível conectar ao copiloto. Tente novamente em instantes." |
| Tool call failed (DB query error) | Inline assistant message | "Não consegui buscar esse dado agora. Tente reformular a pergunta ou tente novamente." |
| Rate limit (429) | Inline assistant message | "Muitas perguntas em sequência. Aguarde alguns segundos e tente novamente." |

Error messages are shown as a MessageBubble with `role="assistant"` variant but with `text-destructive` foreground on the text only. No special background.

### Read-Only Enforcement (D-05)

The copilot sidebar contains NO action buttons that mutate data. There are no "Confirmar", "Remarcar", "Gerar cobrança" buttons inside the panel. If the user asks the copilot to perform a write action, the assistant responds with a guidance message pointing to the relevant UI section (e.g., "Para remarcar, acesse a Agenda e clique no evento."). The Route Handler enforces this at the tool layer — only read tools are registered.

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Trigger button (icon-only) | `aria-label` toggled between "Abrir copiloto" / "Fechar copiloto". `aria-expanded` state on button element. |
| Sidebar panel | `Sheet` from shadcn uses `role="dialog"` + `aria-modal="true"` automatically via Radix Dialog primitive. |
| Message list live region | `MessageList` `<div>` must have `aria-live="polite"` + `aria-label="Conversa com o copiloto"`. New messages announced as they arrive. |
| Streaming cursor | `<span aria-hidden="true">` — purely decorative, must not be read by screen readers. |
| Suggested prompt chips | Each chip is a `<button>` with visible label text. No icon-only chips. |
| Input field | `<textarea aria-label="Mensagem para o copiloto" />`. Required for screen readers (no visible `<label>` due to placeholder-only pattern). |
| Send button | `<Button aria-label="Enviar mensagem" disabled={isLoading || !input.trim()}>Perguntar</Button>`. |
| Typing indicator | `<div role="status" aria-label="Copiloto digitando…">` — announced once to screen readers. |
| Focus management | When Sheet opens, focus moves to the input field. When Sheet closes, focus returns to trigger button. (Sheet default behavior — verify Radix Dialog `initialFocus` prop.) |
| Keyboard: send on Enter | `onKeyDown` handler on textarea: `if (e.key === 'Enter' && !e.shiftKey) { e.preventDefault(); handleSubmit(); }` |

## AI SDK Integration Contract

This section is for the executor — it defines how `useChat` maps to the visual contract.

| AI SDK hook | UI binding |
|-------------|------------|
| `useChat({ api: '/api/copilot' })` | Mounted in `CopilotSidebar.tsx` |
| `messages` | Mapped to `MessageList` → `MessageBubble` for each entry |
| `input` | Bound to `CopilotInput` textarea value |
| `handleInputChange` | onChange on textarea |
| `handleSubmit` | onSubmit on form / Enter key handler / chip click |
| `isLoading` | Controls typing indicator visibility + send button + input disabled state |
| `setMessages([])` | Called by "Limpar conversa" button |
| `error` | Rendered as an assistant MessageBubble with `text-destructive` |

**Route handler contract (`/api/copilot`):**
- Method: POST
- Runtime: `export const runtime = 'nodejs'`
- Input: standard AI SDK message format (JSON body)
- Model: `anthropic/claude-sonnet-4-6` via Vercel AI Gateway (`AI_GATEWAY_API_KEY` read at call-time)
- Output: streaming text response (AI SDK `streamText` → `result.toDataStreamResponse()`)
- Auth: read user session from Supabase SSR cookie; reject 401 if unauthenticated
- Tools: only read-only, tenant-scoped tools registered (no write tools — D-05)
