## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-collect-T | Tampering | LLM hallucinates payment link | mitigate | link ALWAYS from gateway.getInvoiceUrl(providerChargeId); null → abort send; LLM system prompt forbids URLs; test asserts no hardcoded link literal (D-04) |
| T-5-collect-I | Information Disclosure | PII to LLM in personalization | mitigate | only first name + amount sent (RESEARCH Pattern 6); ZDR zeroDataRetention; no CPF/health |
| T-5-collect-lgpd | Information Disclosure | message to deleted/anonymized patient | mitigate | scan filters deleted_at IS NULL + is_anonymized=false (mirror ruler WR-06) |
| T-5-collect-tenant | Information Disclosure | cross-tenant read on agent log page | mitigate | page uses createClient() — RLS SELECT policy `tenant_id = get_my_tenant_id()` (05-01) |
| T-5-collect-cron | Spoofing/EoP | unauthorized cron trigger → mass sends | mitigate | isCronAuthorized fail-closed bearer check → 401 |
| T-5-collect-dup | Tampering | duplicate sends on cron retry | mitigate | outbox idempotencyKey `collection-agent:{receivableId}:{date}` UNIQUE |
