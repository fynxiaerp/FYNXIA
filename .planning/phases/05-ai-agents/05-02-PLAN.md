## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→/api/copilot | Authenticated staff chat input crosses to server |
| copilot tools→Supabase | RLS-scoped reads under user session |
| server→AI Gateway | Tool results (masked) leave to LLM provider |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-pii | Information Disclosure | tool results → LLM provider | mitigate | maskCPF + maskPhone + maskEmail; NEVER select medical_history/allergies/medications/anamnese; ZDR `zeroDataRetention: true` (D-01/D-02) |
| T-5-tenant | Information Disclosure | cross-tenant tool data | mitigate | tools use `createClient()` (user session) — RLS enforces `tenant_id = get_my_tenant_id()`; createAdminClient banned in tools.ts |
| T-5-eop | Elevation of Privilege | prompt injection → write action | mitigate | D-05 — only read tools registered; tools contain no insert/update/delete/upsert; RLS read-only regardless of prompt |
| T-5-auth | Spoofing | unauthenticated copilot use | mitigate | `supabase.auth.getUser()` gate → 401 before any tool runs (V2) |
| T-5-secret | Information Disclosure | AI_GATEWAY_API_KEY exposure | mitigate | server-only, read at call-time, never NEXT_PUBLIC_ |
| T-5-input | Information Disclosure | user types PII into chat | accept | system prompt instructs no-repeat of CPF/health; ZDR prevents retention; documented limitation (RESEARCH Pitfall 10) |
