<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| uploaded file → vision model | Untrusted file bytes leave to the Gateway; ZDR mandatory; MIME+size validated |
| extracted fields → patient cadastro | Model output is untrusted until a human reviewer confirms; reviewer fields are authoritative |
| reviewer session → ocr_extractions | RLS tenant-scoped; assertNotReadOnly blocks read-only roles |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-20 | Information Disclosure | PII (RG/CPF) to vision model | mitigate | zeroDataRetention:true on every Gateway call; source-inspection enforced |
| T-10-21 | Information Disclosure | raw CPF in logs | mitigate | maskCPF before any console; ocr.committed logs IDs only |
| T-10-22 | Tampering | malicious file upload | mitigate | MIME allowlist (image/pdf) + 4MB size guard → 415/413 |
| T-10-23 | Tampering | bad extraction written without review | mitigate | below-threshold → status pending_review; commit requires human confirm via Server Action |
| T-10-24 | Information Disclosure | cross-tenant extraction read | mitigate | ocr_extractions RLS USING get_my_tenant_id(); actions scope clinic_id from actor |
| T-10-25 | Spoofing | unauthenticated OCR call | mitigate | auth.getUser() gate → 401 |
</threat_model>
